# OmniAI Search 浏览器插件 - 产品开发文档

## 1. 产品概述
* **产品名称**：OmniAI Search（暂定）
* **产品形态**：浏览器插件（主要支持 Chrome、Edge 等主流浏览器）
* **核心目标**：在用户使用传统搜索引擎（如 Google、Bing、百度）时，在搜索结果页面的旁边直接展示 AI 生成的回答。将传统搜索的“广度”和 AI 的“深度”结合，大幅提升用户找资料的效率。

## 2. 核心功能
* **搜索增强（对比参考）**：用户在主流搜索引擎（支持 Google、Bing、Yahoo、Baidu、DuckDuckGo、Kagi、Yandex、NAVER、Brave、SERX 等）输入问题时，插件会自动在网页右侧空白处生成一个卡片，展示 AI 给出的直接答案。
* **多 AI 模型支持与管理**：
  * 支持市面上主流的 AI 工具，用户可以在设置中手动勾选需要显示的 AI 平台。
  * **默认勾选**：豆包、千问、ChatGPT、DeepSeek。
  * **支持的 AI 列表及入口地址**：
    * ChatGPT (https://chatgpt.com)
    * Gemini (https://gemini.google.com)
    * 豆包 (https://www.doubao.com)
    * 千问 (https://www.qianwen.com)
    * Kimi (https://www.kimi.com)
    * Claude (https://claude.ai)
    * Perplexity (https://www.perplexity.ai)
    * DeepSeek (https://chat.deepseek.com)
    * Grok (https://grok.com)
    * 智谱清言 (https://chatglm.cn)
    * Z.ai (https://z.ai)
    * Copilot (https://copilot.microsoft.com)
* **独立窗口打开**：每个 AI 对话窗口都提供一个“在网页打开”的按钮，点击后可直接跳转到对应 AI 的官方网页继续对话。
* **对话收藏**：支持一键收藏有价值的 AI 对话内容，方便后续查找和回顾。
* **主题外观切换**：支持界面主题切换，提供“自动（跟随系统）”、“明亮模式”、“暗黑模式”三个选项，默认设置为“自动”。
* **网页总结（效率提升）**：提供一键提取当前网页核心内容的功能，并利用内嵌的 AI 网页窗口对内容进行快速总结（需探索自动填入长文本的方法）。
* **侧边栏对话（无缝体验）**：在浏览器屏幕边缘放置一个悬浮球，点击展开由插件原生提供的侧边栏（不使用 iframe 嵌套，而是插件自己的 UI）。遇到不懂的词或想追问细节时，直接在当前网页通过侧边栏提问，实现无缝对话体验。
* **网页广告拦截（可选功能）**：提供基础的网页广告屏蔽功能，用户可以在设置中心（或侧边栏小窗）手动开启或关闭，提升整体搜索与浏览的清爽度。

## 3. 技术方案与开发环境
* **开发设备**：MacBook Pro 14 (M1 芯片)
* **开发语言**：HTML、CSS、JavaScript
* **核心技术路线 (API 直连方案)**：
  * **背景请求 (Background Fetch)**：放弃早期的 iframe 嵌套方案，改用在插件后台直接发送 API 请求，获取毫秒级响应。
  * **流式渲染 (Server-Sent Events)**：接收大模型逐字返回的数据，并在原生 UI 中实现类似打字机的流畅显示。
  * **内置与自定义 API Key**：插件内置一个由开发者提供的 OpenRouter 公共 Key，用户安装即可免费使用（开箱即用）。同时保留设置入口，允许高阶用户填入自己的 Key，以防公共 Key 被限流。

## 4. 界面设计与 UI 风格规划
为了保证插件既美观又不会喧宾夺主（不干扰原本的搜索结果），我们将采用以下 UI 设计风格和规范：

### 4.1 UI 风格基调 (现代极简风格)
* **轻量化卡片 (Lightweight Cards)**：为了保证网页滚动的绝对流畅（避免毛玻璃等复杂滤镜可能带来的性能卡顿），侧边栏和悬浮卡片采用纯色背景（明亮模式下为纯白或极浅灰，暗黑模式下为深灰），配合微妙的边框和柔和的阴影（Soft Drop Shadow）来区分层级，营造“悬浮”的视觉效果。
* **极简排版 (Minimalism)**：去除多余的边框和复杂的装饰，使用大圆角（例如 `border-radius: 12px` 或 `16px`），让界面看起来更友好，让内容成为绝对的主角。
* **微交互动画 (Micro-interactions)**：悬浮球的展开、按钮的悬停、主题的切换，都加入平滑的过渡动画（如 0.2s-0.3s 的 ease-in-out），让操作感觉“丝滑”、“跟手”。
* **高对比度文字**：在明亮/暗黑模式下，严格控制文字与背景的对比度，确保用户长时间阅读 AI 长文本时眼睛不会疲劳。

### 4.2 核心组件设计与布局
* **结果注入区 (搜索页旁侧小窗)**：
  * **整体布局**：采用经典的“顶部操作栏 + 核心内容区 + 底部扩展区”的上下结构，宽度固定（约 350px-400px，确保不挤压原网页主体），高度自适应内容，最大高度不超过屏幕可视区域的 80%。
  * **顶部操作栏 (Header)**：高度精简（约 40px），包含：
    * **左侧**：AI 模型切换下拉菜单（带各 AI 的小图标，如下拉选择“豆包”、“ChatGPT”）。
    * **右侧**：三个小图标按钮依次为“收藏当前对话”、“在网页打开（跳转官网）”、“设置/关闭”。
  * **核心内容区 (Content - 嵌套 iframe)**：占据小窗 90% 的面积，直接展示内嵌的 AI 官方网页。通过 CSS 隐藏掉该 AI 网页本身多余的侧边栏或导航，只保留最纯粹的对话流。
  * **视觉边界**：小窗外围采用 1px 的极细浅色边框和柔和的底层阴影与主网页隔离。

* **悬浮球与侧边栏 (独立对话区)**：
  * **悬浮球**：一个圆润的图标，贴在屏幕右下角边缘。
  * **侧边栏布局**：点击后从屏幕右侧滑出（宽度约 400px，高度 100% 撑满屏幕）。
    * **顶部**：标题（如“AI 助手”）与关闭按钮。
    * **主区**：完全由插件原生的对话气泡流组成（用户提问靠右，AI 回答靠左）。
    * **底部**：固定在底部的输入框区域，包含文本框、“网页总结”快捷按钮和发送按钮。

* **设置中心**：一个独立的配置页面，采用“左侧导航菜单 + 右侧内容面板”的经典布局，包含：
  * **AI 平台显示设置**：列出所有支持的 AI 及入口地址，提供复选框，默认勾选豆包、千问、ChatGPT、DeepSeek。
  * **搜索面板配置**：提供各个搜索引擎（Google、Bing、Baidu、Yahoo 等）的启用/禁用开关。
  * **外观主题设置**：提供自动、明亮、暗黑三个选项。
    * **广告拦截开关**：提供全局开启/关闭广告屏蔽的选项。
    * **收藏夹**：集中展示用户收藏的对话内容。

## 5. 开发阶段划分 (实施路线图)
为了避免一开始觉得太复杂，我们将开发分为四个循序渐进的阶段：

* **第一阶段：跑通基础流程（打地基）**
  * 配置 M1 Mac 上的开发环境。
  * 创建基础的插件文件，能安装到你的浏览器里。
  * 实现在 Google 搜索页面旁边“塞入”一个包含豆包（或 ChatGPT）网页的小窗（iframe）。
* **第二阶段：完善基础功能与多平台支持**
  * 将所有收集到的主流 AI 工具入口地址写入配置中。
  * 开发“设置中心”页面，实现：AI 平台勾选（默认四项）、搜索引擎开关配置、主题颜色切换（自动/明亮/暗黑）。
  * 实现根据用户的搜索词自动填充到内嵌 AI 窗口的输入框并发送。
* **第三阶段：高级交互与体验升级**
  * 在卡片上方增加“在网页打开”按钮，点击跳转对应 AI 官网。
  * 添加“收藏对话”功能，并能在设置中心查阅。
  * 完善各种搜索引擎页面的适配测试（确保 Google、Bing、Baidu 等页面不发生排版冲突）。
* **第四阶段：侧边栏与扩展功能开发**
  * 开发插件原生的侧边栏 UI，实现悬浮球唤醒。
  * 评估并引入基础的广告拦截规则列表（如 EasyList），实现广告屏蔽的开关功能。
* **第五阶段：优化与打磨（精雕细琢）**
  * 美化卡片和侧边栏的界面（CSS 样式调整）。
  * 修复可能出现的显示错误（Bug），准备最终打包。
